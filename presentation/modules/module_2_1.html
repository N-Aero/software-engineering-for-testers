Rule 2: Don't use the ELSE keyword

Else can produce an additional nesting level which makes your code harder to read. Usually this can be solved by early returning.

  • Else can reduce deep nesting levels, which makes it hard to follow
  • This can be solved by first getting rid of the "invalid" cases.

Rule 3: Wrap all primitives and Strings

  • When a method takes an int as a parameter, the method name needs to do all of the work of expressing the intent. If the same method takes an Hour as a parameter, it’s much easier to see what’s going on.
  • With a primitive variable the compiler can’t help you write semantically correct programs.
