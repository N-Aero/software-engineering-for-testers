Why is clean code important?

  • "There will be code"
    - Robert C. Martin
  • More readers than authors.
  • Understandability affects the cost of change
  • Working in a clean environment affects your state of mind.
  • Code will always exist. It is suggested that nobody has to write code in the future, but this has been posed years ago. The reality shows that the need for developers, and thus the production of software increases. And therefor, we should manage this. Remember that code is really the language in which we ultimately express the requirements
  • As they added more and more features, the code got worse and worse until they simply could not manage it any longer. It was the bad code that brought the company down. Code has a lifecycle, it is written down, then read by others to understand it so a system can change. Even when that specific line is changed, it has to be read before it is changed. Sense making of existing code is a one of the most important prerequisites Of course this only refers to meaningful code, that is written to make money or make the world a better place.
  • Being able to single-handed change a piece of code in a system without breaking it is very valuable. Waiting for others to change it (for example the system owner) make you wait, and thus costs time. Introducing a bug might me more costly.
  • Clean spaces help you feel welcomed.

Martin's book is one of the most referred literature regarding good code. 'Clean' is a well-known term in the industry nowadays. The main points from his book are the points of interests mentioned here. They are more or less sorted by precedence. For example, readability is more important than less code.
